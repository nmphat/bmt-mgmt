## Critical Gaps

### ADMIN-02 — Create session RPC mismatch

- **Flow:** Admin creates session.
- **Breaks at:** Create session form submit.
- **Frontend evidence:** `src/views/CreateSessionView.vue` calls `supabase.rpc('create_session_with_intervals', ...)`.
- **Current backend evidence:** `docs/sql-export/06_functions.sql` and production metadata expose `create_session_with_bookings(...)`, not `create_session_with_intervals`.
- **Impact:** Admin create-session E2E can fail at submit.

### ADMIN-03 — Member CRUD sends removed `is_permanent`

- **Flow:** Admin creates or edits member.
- **Breaks at:** Supabase insert/update payload.
- **Frontend evidence:** `src/views/MemberView.vue` includes `is_permanent` in new-member state, insert payload, edit payload, and UI.
- **Current backend evidence:** `docs/sql-export/02_tables.sql` and production metadata show `public.members` columns `display_name`, `role`, and `is_active`, but no `is_permanent`.
- **Impact:** Admin member add/edit can fail against the current schema.

### SESS-03 — Session edit sends non-table `court_fee_total`

- **Flow:** Admin edits session.
- **Breaks at:** Supabase update payload.
- **Frontend evidence:** `src/views/SessionDetailView.vue` updates `sessions.court_fee_total`.
- **Current backend evidence:** `docs/sql-export/02_tables.sql` and production metadata show `sessions.court_fee_addon`, `price_per_hour`, and `shuttle_fee_total`, but no `court_fee_total`.
- **Impact:** Admin session edit can fail at save.

## Tech Debt and Warnings

- Manual visual UAT and 360/390/430px sweeps were user-skipped during phase execution and replaced by automated/source/browser evidence.
- `HomePage.vue` has non-blocking payment/RPC console logging.
- Historical Phase 2 validation/planning text still mentions `is_registered_not_attended`; current source/docs use `interval_presence` only, so this is documentation debt rather than a current runtime blocker.

- Payment polling has duplicated paths (`check_qr_status` composable vs direct snapshot polling in `PaymentQRModal.vue`).
