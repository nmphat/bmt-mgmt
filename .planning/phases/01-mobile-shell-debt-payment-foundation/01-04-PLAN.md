<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| QR sheet -> Supabase snapshot polling | Client reads payment status while sheet is open. |
| guest QR action -> payment display | Guest can view/copy payment transfer content but cannot perform admin mutation. |
| admin cash input -> `add_manual_payment` RPC | Admin-entered cash amount/note crosses from UI into backend mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-04-01 | Tampering | `PaymentQRModal.vue` amount/content | mitigate | Use backend `snapshot` or `groupData`; no frontend authoritative debt calculation. |
| T-01-04-02 | Denial of service | QR polling | mitigate | Poll only while open; stop on close and unmount. |
| T-01-04-03 | Repudiation | Manual cash confirmation | mitigate | Require review step showing member/session, amount, remaining debt, note before RPC call. |
| T-01-04-04 | Elevation of privilege | Manual cash controls | mitigate | Do not add guest cash entry points; preserve caller-side admin gates. |
| T-01-04-05 | Information disclosure | Payment transfer content | accept | QR transfer amount/content are intentionally visible for payment completion. |
</threat_model>
