<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| guest input -> Supabase debt query | Search text is untrusted input passed to query filters. |
| UI selected totals -> payment RPC | UI selected total is display-only; RPC/backend owns grouped payment amount. |
| guest QR action -> payment sheet | Guests can create QR flows but cannot mutate cash/admin payment state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-03-01 | Tampering | `HomePage.vue` group payment flow | mitigate | Continue using `create_group_payment` RPC result for authoritative amount/code. |
| T-01-03-02 | Information disclosure | `HomeDebtTable.vue` cards | mitigate | Home cards show approved summary fields only; session-level rows remain on member detail. |
| T-01-03-03 | Denial of service | Debt search | mitigate | Keep paginated query/load-more; reset page on search; no unbounded infinite scroll. |
| T-01-03-04 | Elevation of privilege | Guest QR actions | accept | Guest QR generation is approved; no cash/admin controls are added. |
</threat_model>
