<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| guest browser -> shell navigation | Public navigation exposes only approved read-only destinations. |
| authenticated user -> header admin controls | Admin controls must require `authStore.isAdmin`. |
| client UI -> hidden login route | `/login` remains available by direct URL but not advertised to guests. |
| fixed mobile layers -> user actions | Bottom nav, group bar, sheets, and toasts must not obscure primary actions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-02-01 | Information disclosure | `BottomNav.vue` | mitigate | Bottom nav contains only Home/Debt, Members, Sessions; no Login/Admin/Create links. |
| T-01-02-02 | Elevation of privilege | `AppHeader.vue` | mitigate | Admin link appears only when `authStore.isAdmin`. |
| T-01-02-03 | Spoofing | Language/auth menu controls | mitigate | Preserve existing Pinia auth state and Supabase signOut flow; no custom auth state. |
| T-01-02-04 | Denial of service | Fixed bottom nav overlap | mitigate | Add safe-area content padding and verify 360/390/430px in final validation. |
| T-01-02-05 | Denial of service | Toast/fixed-layer overlap | mitigate | Define layer order and toast offset/top placement so toasts do not block bottom nav, group CTA, or sheet footer actions. |
| T-01-02-06 | Denial of service | Logout redirect/fetch loop | mitigate | Preserve Supabase signOut cleanup, redirect to `/`, and guard authenticated debt fetching after logout. |
