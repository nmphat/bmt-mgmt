<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| guest browser -> public routes | Unauthenticated users can read approved public screens. |
| guest/admin UI -> Supabase anon/auth role | Client-side visibility gates must not imply backend authorization. |
| authenticated non-admin -> admin mutation controls | Non-admin authenticated users must not see admin-only mutation actions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-01 | Elevation of privilege | `MemberView.vue`, `DashboardView.vue` | mitigate | Use `authStore.isAdmin` for member CRUD and Create Session visibility/handlers. |
| T-01-02 | Information disclosure | `router/index.ts` | accept | `/`, `/members`, `/member/:id`, `/sessions`, `/session/:id` are intentionally public read-only per locked decisions. |
| T-01-03 | Tampering | Member/session mutations | transfer | Backend RLS/Supabase policies remain authoritative; UI gates reduce accidental exposure only. |
| T-01-04 | Repudiation | Preservation matrix | mitigate | Record route/access/payment contracts before implementation so later verification can compare intended behavior. |
</threat_model>
