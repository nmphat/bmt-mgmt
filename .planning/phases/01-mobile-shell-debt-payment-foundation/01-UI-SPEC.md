# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — use app-owned Vue/Tailwind primitives |
| Preset | not applicable |
| Component library | none; create/reuse local Vue primitives only |
| Icon library | `lucide-vue-next` |
| Font | system sans: `-apple-system, BlinkMacSystemFont, "SF Pro Text", "Segoe UI", system-ui, sans-serif` |

Source: `01-RESEARCH.md`, `brand-spec.md`, `package.json`, `src/assets/main.css`.

Design-system rule: do not introduce shadcn or another third-party UI framework in Phase 1. Build with Vue 3, TailwindCSS v4 utilities, existing i18n, and local components.

---

## Spacing Scale

Declared values must be multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, checkbox/label micro gaps |
| sm | 8px | Compact element spacing, chip/button inner gaps |
| md | 16px | Default card padding, page side padding on mobile |
| lg | 24px | Section gaps, sheet header/content spacing |
| xl | 32px | Major card groups, page blocks |
| 2xl | 48px | Desktop section rhythm only |
| 3xl | 64px | Desktop page-level spacing only |

Exceptions:
- Minimum interactive target: `44px` height/width for buttons, icon buttons, nav tabs, search input, sheet close.
- Mobile sheet cap: `max-height: 88dvh`.
- Fixed bottom nav offset: `16px + env(safe-area-inset-bottom)`.
- Sticky group payment bar bottom offset: `92px + env(safe-area-inset-bottom)`.
- Pages with bottom nav reserve bottom padding of at least `96px + env(safe-area-inset-bottom)`.
- Pages where group bar can appear reserve bottom padding of at least `148px + env(safe-area-inset-bottom)`.

---

## Typography

Use exactly these four mobile text sizes for changed Phase 1 UI. Existing enlarged Tailwind theme may remain globally, but changed mobile components must intentionally apply density classes to match this contract.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label / Meta | 14px | 700 | 1.35 |
| Heading | 20px | 700 | 1.2 |
| Display / Debt amount | 32px | 700 | 1.05 |

Allowed weights: `400` and `700` only.

Rules:
- Debt totals use tabular numerals where practical.
- Vietnamese labels must not be truncated unless the full value is available through context, title, or visible detail text.
- Button text uses 14–16px at weight 700.
- Avoid all-caps except short meta labels/status chips.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(98.5% 0.002 247)` / Tailwind `gray-50` | App background and neutral shell |
| Secondary (30%) | `oklch(100% 0 0)` / white | Cards, sheets, bottom nav, header |
| Accent (10%) | `oklch(51.1% 0.262 276)` / Tailwind `indigo-600` | Primary QR CTA, selected checkbox/card border, active bottom nav tab, focus ring, Details links |
| Destructive | Tailwind `red-600` | Destructive actions only: delete member, cancel destructive confirmation |

Accent reserved for:
- `Tạo QR thanh toán` / `Create payment QR`
- `Tạo QR gộp` / `Create group QR`
- selected debt-card border/checkmark
- active bottom-nav tab
- focus rings for inputs/buttons
- explicit Details links/buttons

Do not use accent for every interactive element. Secondary buttons remain white or indigo-soft.

Semantic status colors:
- Open: blue
- Waiting/payment pending: orange
- Done/paid/debt-free: green
- Cancelled/neutral/disabled: gray
- Destructive delete/cancel: red only when the action is destructive

## Phase Interaction Contract

### Mobile Shell

- Bottom nav appears on mobile and has exactly three public tabs:
  1. Home/Debt: `/`
  2. Members: `/members`
  3. Sessions: `/sessions`
- No Admin tab.
- No visible guest login button.
- `/login` remains manually addressable for admins.
- Language switcher is always visible in the compact header for guest and admin.
- After login, profile/logout/admin actions appear only in compact header/menu.
- Bottom nav uses fixed positioning with safe-area padding and must not cover content at 360px, 390px, or 430px.

### Header

- Header is compact and sticky.
- Primary public navigation moves to bottom nav on mobile.
- Header right side order:
  1. Language switcher
  2. Authenticated debt badge, if logged in
  3. Profile/admin/logout menu, if logged in
- Guest header must not show login.

### Debt Home

- Debt is the primary content on `/`.
- Data source remains `view_member_debt_summary`.
- Support member-name search.
- Keep total-debt sorting.
- Keep load-more button behavior.
- Do not add overdue, high-debt, or partial-payment chips in Phase 1.
- Loading uses skeleton debt cards, not only a spinner.
- Empty state uses debt-free copy from this contract.

### Debt Cards

Each mobile debt card shows:
- member name
- total debt
- unpaid session count
- selected state
- QR action
- explicit Details link/button

Interaction:
- Tapping card body toggles selection.
- Checkbox also toggles selection.
- Only Details navigates to `/member/:id`.
- QR button creates payment for that member.
- No per-session rows on the home card.
- No long-press selection in Phase 1.

### Sticky Group Payment Bar

- Appears only when one or more debt members are selected.
- Sits above bottom nav, not at `bottom: 0`.
- Displays selected count and selected total.
- CTA label: `Tạo QR gộp` / `Create group QR`.
- Must not overlap debt cards, load-more button, bottom nav, toast, or sheet trigger at 360px, 390px, or 430px.

### Sessions List

- `/sessions` is public read-only.
- Session cards show title, date/time, status, interval count, registration count, and Details action.
- Create Session is visible only for `authStore.isAdmin`.
- Guests must not see create/edit/delete/finalize/cash/admin actions.

### Members List

- `/members` is public read-only for guests.
- Guests can search/view members and navigate to member detail.
- Admins can add/edit/delete members after login.
- Mutation controls should use `authStore.isAdmin`, not only `isAuthenticated`.

### Member Detail / Debt History

- Data source remains `view_member_session_details`.
- Preserve Pay All / group QR and single-session QR entry points.
- Do not calculate authoritative debt amounts in frontend.
- Member detail may show session-level rows because this route is the detail view.

## Payment Sheet Contract

### Responsive Presentation

- Mobile: bottom sheet.
- Desktop: centered dialog.
- Mobile sheet max height: `88dvh`.
- Sheet body scrolls internally.
- Footer actions are sticky and safe-area-aware.
- Close button remains visible and 44px tappable.
- Overlay click may close only when no submission is in progress.

### QR Sheet Order

Pending QR content order is fixed:

1. Amount first
2. QR image
3. Transfer content + copy action
4. Member/group breakdown
5. Status note/actions

### Polling and Success

- Poll only while sheet is open.
- Stop polling on close/unmount.
- On paid:
  - show success state
  - emit refresh
  - do not auto-close
  - user closes with Done/Close

### Guest Payment Restrictions

- Guests see QR payment actions only.
- Guests never see cash/manual payment actions.
- No guest “I transferred” confirmation in Phase 1.

### Admin Cash

- Cash appears only for admin and only where existing snapshot context safely supports `add_manual_payment`.
- If cash sheet is touched in Phase 1, it must include double confirmation:
  1. amount/note entry
  2. review screen with member/session, amount, remaining debt, and note
- Call `add_manual_payment` only after the second confirmation.

## Accessibility and Mobile Ergonomics

- Minimum target size: 44px.
- Search input min height: 48px.
- Sheet close, nav tabs, QR, Details, load-more, and group CTA must be keyboard/focus visible.
- Bottom sheet uses `role="dialog"` and `aria-modal="true"`.
- QR image keeps meaningful alt text.
- Toasts must not cover sheet footer or group payment CTA.
- Use `aria-live` for payment success/status where practical.
- Selection state must be visually clear beyond color alone: checkbox/checkmark plus border/background.

## Source-of-Truth Preservation

Do not change these behavior contracts:

| Area | Source |
|------|--------|
| Guest debt summary | `view_member_debt_summary` |
| Member debt history | `view_member_session_details` |
| Group QR | `create_group_payment` |
| QR amount/code | `snapshot` or `groupData` from backend/RPC |
| Manual cash | `add_manual_payment` |
| Route access | `src/router/index.ts` |
| Auth/admin state | `src/stores/auth.ts` |
| Language | `src/stores/lang.ts`, `src/locales/messages.ts` |

Frontend must not recalculate shared costs, split fees, or derive authoritative payment amounts.
